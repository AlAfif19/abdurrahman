# Product Requirement Document

## Project Name

Split Personality Portfolio Website — Al Afif Abdurrahman

---

# 1. Overview

Website portfolio statis personal yang menampilkan identitas profesional dan personal dalam dua mode tampilan:

- Professional Mode
- Casual Mode

Website dirancang untuk menampilkan profil, skill, pengalaman, project, organisasi, dan pencapaian secara modern, interaktif, dan responsif.

---

# 2. Goals

## Primary Goals

- Menampilkan portfolio profesional untuk recruiter dan HR
- Menunjukkan skill teknis di bidang AI/ML dan software development
- Menampilkan personal branding yang kuat
- Memberikan pengalaman visual yang modern dan interaktif

## Secondary Goals

- Menampilkan sisi personal/non-teknis pengguna
- Meningkatkan daya tarik visual portfolio
- Menjadi showcase project frontend modern

---

# 3. Target Users

- HR / Recruiter
- Tech Lead / Engineering Manager
- Hiring Manager
- Client
- Networking / Community

---

# 4. Core Features

## 4.1 Theme Switch

User dapat mengganti tampilan:

- Professional
- Casual

Animasi transisi harus smooth.

## 4.7 Organization Section

Menampilkan pengalaman organisasi.

# 5. Non Functional Requirements

- Responsive
- Fast Loading
- Smooth Animation
- Mobile Friendly
- Clean Architecture
- Maintainable Code

---

# 6. Performance Requirements

Target:

- Lighthouse Score > 90
- First Load < 3s
- Optimized Assets

---

# 7. Constraints

- Static Website
- No Database
- No Authentication
- No Docker
- Single Command Run
